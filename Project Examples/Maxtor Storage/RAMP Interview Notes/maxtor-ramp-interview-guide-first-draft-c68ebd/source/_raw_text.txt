Introduction:  Use of Storage.  We seek to understand how storage is used in enterprises.  To start out we need to understand some basics about your use of storage.  
Total Storage.
First, I would like to ask you some general questions about your total storage and your basic storage-related policies.
Question	Answer
    1) Where is your disk storage located - one central location, selected data centers, broadly distributed, etc.?	
    2) How much disk storage do you have in each of the key areas where you have disk storage?  Please differentiate between total space available and space that is actually used.  Also, what is the maximum % full that you want for your disk storage.	
    3) How much do you expect the storage in each area to grow in the next 12 months?	
    4) Do you have specific people assigned full-time to the management of this storage?  If so, how many?  If not, how do you manage this storage?	
    5) How do you break out this storage by type of storage architecture - internal DAS, external DAS, SAN, or NAS?	
    6) What OS (es) do you use with this storage?  Please match up to storage architecture and the amount of storage used for each OS.	
    7) Please describe the disk technology that is used with this storage - SCSI, FC, or ATA; size in GB; and rotational speed?	
    8) Do you notice any performance bottlenecks with the applications that use this storage?  If yes, is that bottleneck CPU, network, or storage-based?  If no, as storage increases, where do you think the first bottleneck might arise?	
    9) What are the availability/ performance criteria that you have in place?  Is this informal or is this part of an SLA?  What are the consequences of higher unavailability than planned? of lower performance?	
    10) What type of tape automation infrastructure do you have in place to support your online disk storage?  Please discuss the number of tape automation products, the number of drives in each, and the number of slots.	
    11) What is the backup window (or is it online backup)?  Are there any issues with backup, such as you are running out of backup window or backup jobs fail to run to completion more often than planned?	
    12) How many generations of backup do you keep?	
    13) How is the data protected other than backup, e.g. RAID, remote mirroring, point-in-time copy?	
    14) What is the retention policy for this data, i.e., what are the policies for keeping a particular piece of data online, such as age, transaction closure, etc.?	
    15) For a piece of data that is available online, how many copies do you have:
        ? On disk
        ? On tape for backup purposes located onsite
        ? On tape for disaster recovery purposes located offsite	
    16) The storage hierarchy represents the different types of storage (such as disk and tape) that you have in your organization. Do you have any other layers of the hierarchy other than high-performance online disk and tape?  If yes what are they?   	
Background to Application-Specific Questions
We need to understand how your key applications use storage and the issues that may arise as a result.  We are going to ask about up to three of your most important applications.  Some of the questions may be the same as when you were asked about your total storage.  The reason that we ask these questions is that the specific application may do something differently than is true generally for the total.  If not simply answer the same.  The first application we will ask about is your primary database application.

Classification of Key Applications (reference data for the interviewer)
Category	Type	Distinguishing Characteristics
Database	Traditional Online Transaction Processing (OLTP) (such as order entry)	Update intensive (both reads and writes, but focus is on writes)
Database	Contemporary OLTP (such SAP/R3)	Mix of writes and reads - updating for transactions, but also random reads
Database	Business intelligence (such as data warehousing)	Query intensive (primarily sequential reads)
File/Database 	Web application	Primarily random reads with a little transaction processing
File	Personal productivity (such as e-mail)	Write once, read once to many times
File	Interactive design (such as CAD/CAM and post production editing)	Mix of reads and writes on large pieces of data
File	Unique large files (such as X-rays and MRIs)	After creation, the content of the file is fixed (in fact, inalterable would be good) and is read only
File	Large file distribution (such as MP3 and video files)	Read-only access of previously-created large bit-mapped files
