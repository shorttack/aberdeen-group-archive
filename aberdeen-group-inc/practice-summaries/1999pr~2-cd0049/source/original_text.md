In the most basic sense, a network operating system is software that resides on a server and 
manages the resources within the network.  It organizes the way information is processed 
among and between all the workstations and peripherals attached to it.
Typically, a NOS operates at the workgroup level, tying together end-user PCs.  As a result, 
an effective NOS brings significant IS benefits:





lights-out” administration.

As noted above, a network operating system is the software that resides on a server and 
manages the resources within the network.  But there are also underlying hardware pieces 
necessary to build the network on which the network operating system runs.
PCs are connected to one another and/or to the server via cabling  twisted-pair, coaxial or 
fiber optic cabling.  For the PC and the server to know how to communicate with each other 
over that cabling, each needs a network interface card, or adapter.
The adapters explain how the PC and server communicate, but the devices must be talking 
the same language for the connection to be successful.  This language is specified in the 
transport protocol.  The most commonly used transport protocols are the Internetwork 
Packet Exchange/Sequenced Packet Exchange (IPX/SPX) and the Transport Control 
Protocol/Internet Protocol (TCP/IP).  IPX/SPX is the protocol used by Novell NetWare, 
while TCP/IP was originally designed to be used by the government and higher-education 
facilities, but has evolved into one of the most commonly used transport protocols in use 
today.
Once the cabling and adapters are in place, the transport protocols and other communication 
facilities are specified by the NOS itself.  This primary NOS component is software that 
resides on the server.  In a traditional network operating system environment, this server is 
also where all the files and network resources are resident or are attached to the network.
The server software controls how users access files and resources in an orderly fashion.  
The client also needs a piece of network operating system software in order to communicate 
with the server.  Some clients need only the correct transport protocols to forge that 
connection — most of these are within environments that use TCP/IP.  But most network 
operating systems have a client piece specific to the server piece that instructs the two 
exactly how to interact.

Technology built upon Intel Architecture-based servers and the drive of the Windows NT 
operating environment to offer more capabilities are the key market forces that are pushing

To get there, the leaders of the NT/Intel server market (Microsoft, Compaq and Intel) along 
with alliance partners are scaling the hardware environment by offering servers that utilize 
more than four processors.  They also are working with Independent Software Vendors on a

environment.  There are therefore four major trends in the Windows NT/Intel server market



management-type hardware monitoring to boosting I/O subsystem bandwidth to offering 
larger Symmetrical Multiprocessing (SMP) architectures;

Consolidation of servers and of the suppliers themselves — e.g., the use of larger, more 
powerful SMP servers;

Advancing the reliability of the Windows NT operating environment and fine-tuning 
applications to provide improved performance on the platform; and

Speeding the scalability, availability and eventually performance of these servers through the 
use of clustering — still very much a work in progress in the NT environment.
Key suppliers include the big four:  Compaq, Hewlett-Packard, IBM and Dell.  There are a 
number of smaller suppliers — sometimes referred to as “second tier” suppliers —

suppliers often are niche suppliers — meaning that they offer products that are to a great 
extent targeted at specific vertical or application markets — unlike the big four suppliers

The Internet.  Some continue to anticipate that today’s dramatic rise in the use of the 
Internet will have a negative impact on the use of NOSes, and, in fact, that the services 
provided by the Internet would replace those provided by NOSes.  Aberdeen finds that, on
