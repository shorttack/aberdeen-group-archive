CONTINUOUS PROCESSING
Stratus defines Continuous Processing as uninterrupted operati<
without loss of data, without performance degradation, and wit]
out special application programming. The Stratus system provit
true continuous processing with a uniquely simple approach thai
eliminates the complexity associated with other vendors' high i
liability systems. Other high reliability systems use software
to detect failures and to back up and recover from a failure.
Complex software is required to clean up bad data and to restoj
the system to a pre-failure state. The use of software to mon:
tor and restore the state of the system takes processing re­
sources, and that overhead increases in direct proportion to t)
on-line workload, thus impacting performance.

The costs of traditional high reliability systems are quite si
nificant, and are not always obvious when undertaking the impl
mentation of such a system. Performance suffers as software
overhead multiplies with the extra processing needed for check
pointing. At times of peak load, performance can become so de
graded that fault tolerant operation has to be suspended. Thu
the advantage of a highly reliable system is lost during the
critical time period of peak load, and if a failure does occur
the recovery is costly and time-consuming.
The Stratus solution for Continuous Processing is based on the
cooperation of three, independent functions. These functions
are:
o Continuous checking.
o Duplexed components.
o Software monitoring and control.
Continuous checking means that the hardware on each board cont
uously checks itself. In most cases, there are two sets of id
tical circuitry on each printed circuit board. Both sets perf
every operation and the results are compared by hardware. Thi
simple approach (for which Stratus has patents pending) makes
possible to detect malfunctions at the instant they occur. Bo
operational and data integrity are maintained without the com­
plexity and overhead of software checkpointing.
The duplexed components such as CPU’s, memories, controllers,
disks are the backbone of continuous processing. Each compone
is totally independent and contains its own checking logic and
diagnostics. By having two identical boards operating in para
lei, continuous processing is achieved. The checking logic in
stantly detects malfunctions and failures. The failed board i
automatically removed from service before erroneous processing
occurs or before data is corrupted. Processing continues on t
duplexed partner.
Maintenance software receives all hardware maintenance inter­
rupts and it determines the cause and nature of the malfunctio
Transient errors result in the board being restarted. Permane
errors result in the board or device remaining out of service,
while an indication of the failure is sent to selected user te
minals. All errors are logged in a system file for later anal
sis by service personnel. This software runs concurrently wit
user applications.
In summary, the Stratus solution to continuous processing is a
combination of hardware and software. Hardware detects malfun
tions at the point at which they occur. Duplex hardware con­
tinues the processing automatically and with no human or progr
intervention. The maintenance software records and analyzes e
rors.

MULTIPROCESSOR TECHNOLOGY
The Stratus/32 is a multiprocessor system whose basic hardware
building blocks are VLSI chips. That is, the processor has be­
come just a component of the system. Many processors (from a
minimum of eighteen to more than a thousand) are used to make a
complete system. This contrasts markedly with the traditional
approach, where systems are based on one or a few custom CENTRAL
processors. Stratus has gained leverage from the millions of
dollars spent by semiconductor companies in their technological
development. This leverage has enabled Stratus to use its own
engineering research and development resources more effectively,
and thus to leapfrog its primary competitor.
