Bay Networks' Switch Node Routing Switch
Tackles Routing Performance Bottlenecks Head-
On
In today's overcrowded networks, routers can be a real bottleneck. The
dilemma: how to relieve the congestion affecting existing routers without
costly router upgrades or migrating to a virtual circuit switched network
based on ATM. Bay Networks' Switch Node Routing Switch ﬁts neatly into
existing network architectures, bringing micro-second latency and wire speed
switching performance to routing.
Changing Network Dynamics
As they mature into enterprise networks, LANs and campus backbones are
experiencing remarkable growth in size and complexity. Increased
processing power, distributed client-server applications and the
proliferation of Internet Protocol and Web-based technologies throughout
corporate networks have subjected servers, end stations and routers to
continually increasing volumes of transactions.
This has resulted in a major shift in network trafﬁc patterns away from
the original 80%-locally-based/ 20%-cross-subnet model. Users are more
and more frequently accessing servers located in subnets other than their
own. Aberdeen research reveals that today's enterprise network trafﬁc is,
in fact, almost the reverse of earlier patterns. Only 20% of the trafﬁc
remains in local segments, while close to 80% crosses the backbone - and
network routers.
Network Trafﬁc Taxes Routers
Routers were introduced into networks to solve trafﬁc problems as well as
to link network segments together. Routers improve overall performance
because they reduce competition for limited bandwidth through
segmentation and intelligent broadcast ﬁltering. However, because of the
magnitude of increase in both transactions and network complexity,
routers today are working overtime. For many networks, they have
become a major performance bottleneck. The options available for
congestion relief have basically been additional network segmentation,
which does not scale well; expensive router upgrades; or the addition of
switches to off-load Layer 2 forwarding.
Bay Networks' Campus Play
Other internetworking suppliers are proposing switching solutions - be
they Layer 3 Switching, Tag Switching, IP Switching or Flow Switching -
that are targeted at solving performance problems by removing multiple
router hops. Aberdeen contends that these solutions, which are based on
migration strategies to high-speed network media, are generally too
complex for campus network environments and are not always relevant.
Bay Networks' answer to the network performance dilemma is the Switch
Node product family. Its ﬁrst member, the Switch Node Routing Switch,
was announced on March 3, 1997. The Switch Node Routing Switch brings
users a compact integration of switching hardware and routing software
that goes beyond the basic capabilities of Layer 3 routers, combining true
IP/IPX routing control with full wire speed performance.
Key Decision Factors
Aberdeen interviews indicate the following elements are critical evaluating
criteria for IT organizations looking at solutions to router bottlenecks:
The easy integration of solutions into existing networks, with minimal
conﬁguration requirements;
The capability to deliver scalable performance from subnet to subnet;
The ability to accommodate emerging real-time applications such as
video; and
Cost.

The Switch Node Routing Switch successfully addresses all these criteria.
Switch Node Architecture
Based on the same BayRS distributed multitasking routing software used
in Bay Network routers, the Switch Node's architecture is designed so that
the data plane serves as the forwarding component and the control plane
functions as the topology component. Separating these two functions
enables both to be optimized for improved performance. Each runs on
separate 1.2 Gbps buses, with scalable data forwarding managed by
distributed processors and system control by a dedicated central processor.
A special feature, IP AutoLearn(tm), permits the Switch Node to
transparently integrate with existing network routers with no re-
conﬁguration of either routers or clients. IP AutoLearn automatically
learns IP forwarding tables, with a separate high-speed CPU dedicated to
learning Layer 2 and Layer 3 addresses.
Switch Node Hardware
The hardware consists of a 5-slot chassis, a CPU Module, and a choice of
Switching Modules that include:
a 16-port 10Base-T Switching Module;
an 8-port 10Base-FL Switching Module; and
a 2-port 100Base-TX Switching Module.
Switch Node interface cards contain advanced, high-speed RISC
processors that can look up addresses on forwarding table caches very
quickly and forward trafﬁc at wire speeds.
Aberdeen Conclusions
Bay Networks has seized a window of opportunity to bring to market a
cost-effective, robust, true Layer 3 switching solution that addresses the
performance needs of campus networks.
Aberdeen considers Bay's approach right on target. While other suppliers
are proposing solutions that are either more suitable for the LAN/WAN
edge of the network or hybrids with some Layer 3 routing capability in
their Layer 2 switches, Bay offers a non-disruptive, here-and-now
alternative that ﬁts snugly with existing network architecture.

### Abstract

Aberdeen analyst Virginia Brooks examines Bay Networks' Switch Node Routing Switch, announced March 3, 1997, as a Layer 3 switching solution for congested campus networks. The study analyzes how enterprise network traffic patterns had reversed from the traditional 80/20 local/cross-subnet model to 20/80, making router bottlenecks critical. Aberdeen assesses the Switch Node's architecture—separating data plane and control plane on dual 1.2 Gbps buses with IP AutoLearn—as a cost-effective alternative to router upgrades or full ATM migration.
