IBM Network Computing
IBM Application Development Product Family
Executive Summary
Successful IS implementation of a network computing strategy begins with the
choice of development tools.
For users, the whole point of migrating to network computing is to create new
competitive-advantage applications and make existing ones more ﬂexible and
scalable. To customize new applications for an enterprise‘s particular needs,
and to allow existing ones to take advantage of the new network-computing
infrastructure, IS developers need development tools and toolsets whose
features and capabilities support rapid, complexity-hiding creation and
deployment of reliable, scalable, distributed, and ﬂexible applications that fully
leverage the network-computing infrastructure.
IS strategists’ key criteria for network computing toolsets are therefore:
scalability (of the development process and of the resulting application, and
especially for oncoming 3-tier enterprise requirements), open ﬂexibility
(including multiple-platform support and integration with existing languages,
protocols, and applications), software-lifecycle and especially automated-
deployment support, and above all support for the new network-computing
technologies such as Internet-application development (e.g., Internet-
middleware support) and distributed-object support.
IBM’s Application Development (IBM AD) product family offers solutions and
services that not only target IS’ key criteria but also allow users to invoke
IBM’s broad array of other network-computing products. IBM AD includes via
IBM’s Visual Age cross-language development tools, its FlowMark workﬂow
creation tool, its Business Process Modeler modeling tool, and its Team
Connection group development management system. These tools are
particularly notable for their comprehensive object-oriented technology
capabilities, their support for IS’ major 3GLs such as COBOL derived from
long IS-programming experience, and IBM’s training and consulting for
enterprise-scale development.
For IBM shops moving to network computing, IBM AD offers a relatively easy
migration path that supports today’s key IS programming languages and adds to
them major new development toolset technologies such as development-
speeding Visual Programming Environments and distributed-object support.
For network-computing customers and prospects, IBM AD offers well-
integrated access to a broad range of IBM network-computing products such as
IBM’s Extended Transaction Services middleware that provides scalability for
Web-enabled data-intensive applications.
Technology Overview: Embedded Experience
Plus New Technologies
IBM AD products (see Table 1) include a suite of Visual Programming
Environments – the VisualAge product family – that provide a Visual
Programming Environment on top of major 3GLs, plus related tools for
program design/analysis (Process Modeller), workﬂow (FlowMark), and group
development (Team Connection). The tools are useful in a broad range of
circumstances, for two reasons: ﬁrst, their ability to combine new programming
technologies with older 3GLs (thus embedding IBM’s COBOL-development-
tool experience in the new toolsets and allowing old-tool users to improve
productivity by upgrading to the new technologies); and second, their support
for particular new technologies key to network-computing-programming
success.

Two of these products are likely to be particularly useful in creating network-
computing applications:
the Visual Generator product that allows relatively automatic generation
and deployment of scalable applications, and is IBM’s equivalent to a
second-generation client-server development environments (CADEs) that
are in the forefront of today’s development-toolset markets and
technologies; and
the upcoming VisualAge for Java product that will allow IBM’s network-
computing users to take advantage of the burgeoning market for Java-
based Internet and Intranet applications.
Below, we describe IBM’s major IBM AD products.
Visual Age
This family of development environments provides a means to visually create
applications from objects. The same Visual Age GUI environment is hosted on
a range of languages: Visual Age for Smalltalk, Visual Age for C++, Visual Age
for COBOL, and Visual Age for BASIC. Thus, developers that in the past have
used IBM tools to code in these languages can now share a common GUI
development tool "look-and-feel".
Relational database support includes DB2/MVS, DB2/400 via the IBM
DDCS/2 gateway, and DB2/6000, as well as non-IBM databases such as
Microsoft SQL Server, Oracle and Sybase SQL Server. Supported
communication protocols include APPC, CPI-C, TCP/IP, Netbios, CICS OS/2,
and EHLLAPI. All IBM AD tools provide support for sharing data via System
Object Model (SOM), Distributed System Object Model (DSOM). and
Microsoft’s Dynamic Data Exchange (DDE).
The Visual Age visual programming environment delivers developer
productivity by providing a familiar integrated development environment,
based on the assembly of parts, across multiple languages and platforms. Visual
Age also provides a set of object libraries that can be used as the basis for
building an application.
A notable member of the VisualAge family is VisualAge for Smalltalk. This
toolset provides a VPE and class libraries for the comprehensive Smalltalk
object-oriented programming language and environment. While Smalltalk itself
has been controversial, this product’s extensive class libraries and IBM’s
support and training provide an exceptionally broad solution for users seeking
to implement and leverage object-oriented technology – and Smalltalk skill sets
are readily transferrable to Java (upcoming VisualAge for Java clients are
targeted to be interoperable with VisualAge for Smalltalk server software).
Visual Age for Java. Internet/Intranet application development will be
supported by a Java-language version of Visual Age that should be released in
1997. The tool will allow developers to visually construct HTML Web pages;
connect Web applications to existing network services; and dynamically
generate HTML pages. Visual Age for Java will also be able to build
applications from (user-interface and infrastructure) parts by drag-and-drop
"visual builds".
Visual Age Generator. Visual Age Generator is a highly productive tool for
generating three-tier transaction-based client server application generation. This
tools delivers a unique advantage to serious cross-platform development efforts
by letting developers generate an application in its entirety in a platform-
independent format (a "speciﬁcation") and then "instantiate" semi-
automatically it into several different execution environments without changing
the speciﬁcation. Visual Age Generator supports clients on OS/2, Windows NT
and Windows 95 platforms. Server support includes Windows NT, OS/2, S/390,
CICS/6000, OS/400, MVS, CICS, VSE, and AIX.
Team Connection. Team Connection delivers the software development
process-management features that are necessary for a group of programmers to

work together and share object code. It also delivers a central repository for
storing and managing the objects shared by developers.
Team Connection provides a persistent object store on OS/2 servers. Tools
services are supported on OS/2, Windows 95 , Windows NT, and Win 3.1
clients. Distribute and build functions are available on Windows NT, OS/2, and
MVS. Distributed build capabilities will be extended to other platforms, such as
CSE, OS/400, AIX, Windows, and variants of Unix.
Team Connection plus VisualAge’s object-assembly approach encourages team
development and the inclusion of the business user in the development process.
Developers can be assigned roles that most closely suit their talents within the
development cycle. Business users can also take part in an interactive and
timely response development loop. Team Connection can ensure a well-
structured application assembly and development process that helps to secure a
robust, bug-free, and scalable result.
DataAtlas. DataAtlas is a performance-analysis and design tool that allows
conceptual modeling of the usage of information by different applications.
Client-server applications run more efﬁciently because DataAtlas tracks data
usage and patterns that it uses to generate a high-performance physical data
design for an application.
FlowMark. Graphical models of the business process are highly effective when
they can be translated into actual physical elements of the network and
application itself. FlowMark acts as the translation mechanism that takes paper-
based thought-processes and turns them into usable workﬂow components.
With FlowMark developers can create deﬁne, document, and test workﬂow
processes.
FlowMark supports OS/2, AIX, Windows NT, and HP-UX servers. Runtime
client support includes OS/2, Windows 3.1, AIX, and Window s NT.
IBM Business Process Modeler (ProModeler). ProModeler graphically maps
the way work moves through an organization. It provides a way to document,
evaluate, and redesign business processes.
ProModeler supports OS/2 and AIX servers with connection to build-time
clients running on OS/2, AIX, and Windows.
A major concern for IS is the need for integration between new competitive-
advantage applications with existing investments and legacy applications. An
integration policy would enable IS to modernize legacy applications without
having to completely rewrite code that works or invest huge sums in an
appropriate hardware and software platform robust and reliable enough to
rehost the old system. During with the process of modernization and redesign
of older applications and processes, IS can identify business process problems
and extract business rules that are embedded in legacy and procedural code.
Extracting these business processes is the ﬁrst step in determining a set of well-
deﬁned business rules and providing an automated and more ﬂexible alternative
to a hard-coded solution.
IBM Business Process Modeler allows IS to identify, design and document
business processes that must be improved, while FlowMark provides a tool for
graphically modeling these processes and turning them into actual automated
workﬂow processes that execute as part of an application or in conjunction with
applications.
Meeting User Criteria
As noted above, IBM AD tools offer particular features that can aid in meeting
users’ network-computing-development needs in the areas of scalability,
ﬂexibility, programmer productivity, software-lifecycle support, and support for
new network-computing technologies.
Scalability. By leveraging IBM’s Extended Transaction Services that can help
to scale data-intensive Internet applications via communications multiplexing
and load-balancing, IBM’s AD tools can help applications to get around key
Web-server Internet bottlenecks (in the IBM Internet application model, the
Java applet can bypass the Web server and connect directly to the network
server supplying the application services via TCP/IP), while DataAtlas also
aids in optimizing data-intensive applications. Team Connection and
ProModeler allow high-level design and group development, helping to scale
the development process for upcoming enterprise 3-tier client-server
requirements.
Flexibility. IBM strongly emphasizes the cross-platform portability of
VisualAge-developed applications and its support for major popular 3GLs.
Several key communication and integration speciﬁcations are supported by
IBM AD and available across the product family to support a distributed
networked environment. IBM SOM allows any applications to talk to one
another and share development objects by supplying a common
communications pipe. A SOM developers’ toolkit allows users to create objects
in one language and access them transparently from another. The OpenDoc
component software architecture provides for the creation of high-level
software objects that can be assembled by end users into applications and

documents. Non-IBM standards for cross-object communication and
integration are also supported, such as DCE, Common Object Request Broker
(CORBA), and the Common Object Services Speciﬁcation.
IBM also emphasizes support for migration of legacy applications to the new
network-computing environments, using toolsets that can "wrap" legacy
COBOL programs as objects that can invoke IBM’s new network-computing
infrastructure capabilities. Likewise, the reusability and modularity of object-
oriented programs means that Smalltalk, C++, and Java programs can be
moved forward effectively as network-computing technologies change.
Programmer Productivity. VisualAge and especially VisualAge Generator offer
VPEs that have been shown to dramatically improve programmer productivity.
IBM’s Smalltalk, C++, and Java support likewise allows users to take
advantage object-oriented-technology’s reusability to increase programmer
productivity.
Software-Lifecycle Support. ProModeler and FlowMark provide sophisticated
design tools, while Team Connection allows some semi-automated application
deployment across remote platforms. VisualAge Generator is a fully-integrated
tool for generating deployable applications from drag-and-drop GUI design.
New-Technology Support. VisualAge for Java will combine with products from
other IBM network-computing product families to deliver a comprehensive
Internet-development solution. These products include MQSeries middleware,
Web-enabled DB2 (or other major popular RDBMSs), Lotus Notes, and
Microsoft’s JDBC Java-gateway product.
For distributed-object support, IBM is offering a basic blueprint (Open
Blueprint) for developing an object technology-based networking scheme. The
IBM Open Blueprint architecture deﬁnes modular services apportioned across
the network and available to applications on all platforms. Thus network
services, distributed system services, application-enabling services, and system
management services would be accessible when needed by any network
applications. The interoperability standards cited above are key parts of the
Open Blueprint.
Visual Age, in particular, offers a graphical and highly productive integrated
development environment for building applications that use Open Blueprint
services. The IBM AD tools’ shared GUI, support for C++ and Smalltalk, and
integration with robust information engineering tools can signiﬁcantly reduce
the effort to build and maintain Open-Blueprint-based distributed applications.
Where IBM AD Tools Are Most Effective In
Network Computing
IBM AD tools are particularly effective in allowing IBM users to move their
present applications towards network computing, e.g., where mainframes are
Web servers or backend database servers. These tools’ support for popular
3GLs such as COBOL and for legacy platforms such as mainframes allow their
users to migrate applications built with these languages and platforms to
network computing relatively swiftly and with relatively low costs and risks.
IBM AD tools also allow IBM customers to begin to take advantage of the new
development technologies associated with network computing, such as VPEs,
distributed-object and repository support for automated software deployment,
and cross-platform portability. Users can also supplement IBM’s tools where
needed with third-party CADEs that provide such features as Web-enabled
dynamic application partitioning. If IBM continues to move its incorporation of
these technologies forward at a rapid pace, IBM AD tools customers may have
the "best of both worlds": exceptional support for legacy applications and
platforms plus the ability to take full advantage of new development
technologies to increase applications’ network-computing scalability, ﬂexibility,
and robust software-lifecycle support, as well as programmers’ development-
process scalability and productivity.
Customers of IBM’s AD tools indicate that these are especially useful in
achieving competitive advantage in moving from past IBM development
products, speeding development, and handling users’ volume and performance
requirements. For example, Amerex cites VisualAge Generator for allowing
rapid migration from CSP with improved productivity and cost savings in their
manufacturing applications, and ImageScan cites VisualAge for C++’s
application-performance scalability and ability to combine object-oriented and
procedural code for check processing solutions.
Aberdeen Conclusions
IBM AD tools provide a solid mix of emerging and tried-and-true development
technologies that help IS signiﬁcantly in creating more ﬂexible, more easily
maintainable, more scalable, and better integrated applications based on
comprehensive networking and bolstered by IBM’s service quality. Object-
oriented technology, support for Java, workﬂow-creation capabilities, and the
visual development paradigm of Visual Age allow IS to bring new features to
the network-computing application and to represent business-critical processes
as components and workﬂows within the application. A broad range of

supported platforms and architectures gives developers more deployment and
development options for building applications and features that will best ﬁt the
individual network environment.
IBM AD tools are a work in progress in an area of rapidly-advancing
technology. Aberdeen anticipates that IBM will move aggressively to deliver
the key VisualAge for Java toolset, and will beef up VisualAge Generator with
such capabilities as a cross-platform application-partitioning-supporting
repository and automated-deployment features as well as better integration with
Web-site authoring tools.
The bottom line is that IBM AD has delivered a toolset that delivers new
capabilities vital to IBM users needing to move ahead in network computing.
Because of development toolsets’ key place in users’ and IBM’s network-
computing strategies, IS buyers should consider IBM AD tools carefully, not so
much for their standalone technology as for their ability to leverage IBM’s
overall network-computing capabilities and users’ legacy applications and
platforms.

## Prescience Detail

### Prediction Assessment: 3/5

Aberdeen's profile demonstrated good understanding of IBM's product direction but missed the transformative Eclipse outcome.

#### Correct Predictions
1. **Java as critical strategic product (CONFIRMED):** Aberdeen identified VisualAge for Java as the most strategically important upcoming IBM AD product. IBM did deliver VisualAge for Java and Java became the dominant enterprise language.
2. **IBM AD primarily valuable for installed-base migration (CONFIRMED):** Aberdeen's characterization that IBM AD tools were valued for their IBM ecosystem integration rather than standalone technology proved accurate.
3. **FlowMark/workflow as key BPM direction (CONFIRMED):** FlowMark evolved into IBM's major BPM product line; workflow/BPM became a major enterprise software category.

#### Missed / Less Accurate
- **Eclipse outcome (MISSED COMPLETELY):** Aberdeen did not anticipate that VisualAge Micro Edition would evolve into Eclipse, donated to open source in 2001 — now the world's most widely used Java IDE. This is arguably the most significant outcome of the entire IBM AD product family.
- **SOM/DSOM viability (INCORRECT):** IBM SOM/DSOM was discontinued; Microsoft DCOM won desktop objects.
- **OpenDoc viability (INCORRECT):** Apple abandoned OpenDoc in 1997; IBM followed. Microsoft OLE/ActiveX won.
- **VisualAge Generator longevity (PARTIALLY CORRECT):** Generator survived (as IBM RDi for IBM i customers) but in a niche role.
