When a PC is Simply Platform Overkill:
IBM’s Network Station Alternative
Preface
For the past ten years, information workers have rightfully demanded PCs on
their desktops in order to:
1. Run the industry’s leading PC-based personal productivity tools —
especially word processing, spreadsheet and electronic mail applications;
2. Take advantage of the graphical user interface capability of client/server
application packages;
3. Access intra-enterprise data located on multiple and varied server types —
including mainframes; departmental servers (AS/400s, Unix, OS/2 Warp, or
proprietary); and/or shared workgroup servers (Novell, LAN Server or NT);
4. Surf to external data and information sources (most recently using the
Internet and Web Browsers for information and data gathering).
But with the use of a new breed of desktop client — the Network Computer
(NC) — information workers no longer require a PC on their desktop to
accomplish these identical tasks. IBM’s new Network Station network
computer allows end users to access information on multiple server types, and
also run PC personal productivity application programs — without forcing the
enterprise to pay for unwanted PC support and upgrade services and unneeded
PC components. Suddenly, placing a full-ﬂedged PC on every desktop may
represent hardware overkill for the majority of information workers.
In this Proﬁle, Aberdeen evaluates IBM’s Network Station — its core
technology, its market positioning, and its value/cost proposition. Aberdeen
concludes that this inexpensive "appliance" is indeed well-positioned to
provide users with access to data and information — wherever it resides —
while providing cost-of-acquisi-tion and cost-of-ownership beneﬁts to the
enterprise as a whole.
Network Station — Providing Experienced Users With The
Tool They Need
Simply stated, IBM’s Network Station is positioned as the stable desktop alter-
native to the Wintel (MS Windows/Intel) PC with the economic and functional
characteristics that experienced IS decision makers have long been demanding.
The majority of information workers need a graphically-oriented desktop
device that can be effectively maintained by IS professionals and allows them
to run what-ever applications and access whatever data they need today and
may need in the future. (See Figure 1.) The enterprises that employ these
individuals want to pro-vide their information workers with a desktop client
that can be cost-effectively supported and upgraded in order that it will not
become obsolete within 2-3 years.

From IBM’s perspective, with both its installed customer base of non-
programmable block mode terminals and its investments into creating a growth
path to web technology-based network computing for its customers, the IBM
Network Station provides a single, elegant, user-oriented tool. Customers with
terminals can feel comfortable that they can upgrade to it without disrupting
their business operations. And IS decision makers who understand that the
current versions of Windows 95, Windows NT, OS/2, and Mac OS will be
obsoleted by their supplier’s next-generation operating systems within the next
18 months can commit to it with minimal risk. As new Java-based applications
and web-based electronic commerce become the norm in the near future, the
IBM Network Station is already fully capable of meeting information workers’
new requirements without any changes.
IBM’s Network Station ﬁlls-the-gap between a native terminal and a full-
ﬂedged PC desktop by providing users with:
Easy access to multiple application environments through the use of 5250,
3270, ASCII, and/or X-terminal emulators. Users merely click on the
appro-priate terminal icon (a hot key) — and, if authorized, are connected
to the application environment of their choice;
Transparent use of client/server and personal productivity environments
— including access to any Windows-based applications (including word
processing, spreadsheets, e-mail, graphics presentations, databases, project
scheduling, etc.) that run on a centralized Windows NT server;
A growth path to new and evolving Java application environments, Java-
based programs, Internet applications, and numerous third-party packaged
applications.
Conversely, NCs are not required by users who perform the same basic,
character-driven task within the boundaries of a constant server environment —
for instance, data input/retrieval, or point-of-sale transaction processing. Users
who perform these tasks under the auspices of a single server usually only
require a simple terminal or display device to do their jobs.
Nor are Network Stations currently designed to fulﬁll the needs of mobile users
(those who live by the laptop) and those information workers who require PC-
grade programmability, display, computational, storage capabilities or require
the attachment of non-standard peripherals. PCs are still needed by end users
who require one or more of the following features:
Local programmability using PC-based programming tools and utilities —
for instance, to write and run complex decision support queries or to
develop software;
Advanced desktop computational and graphical display capabilities —
especially for use in individualized engineering and creative design appli-
cations;
Local storage — for certain branch ofﬁce or home applications — where
remote storage would be inconvenient; and
Specialized peripheral attachment — for function-speciﬁc peripheral
environments that are not currently supported.
From Aberdeen’s perspective, IBM’s Network Stations should explicitly be
used by information workers who would beneﬁt from a highly graphical and
simple interface to access multiple and varied server environments; need access
to character-driven and graphically-oriented applications; and need occasional
access to personal productivity environments such as Microsoft Ofﬁce, Lotus
Notes, etc.

Five elements constitute the design-center for IBM’s Network Station: 1) the
physical systems hardware; 2) the related operating environment; 3) the
communications sub-system; 4) a suite of highly-integrated applications; and 5)
terminal emulators. (See Figure 2.) Note that IS decision makers select the
appropriate monitors to be used in conjunction with their Network Stations.

IBM Network Station — Hardware
The physical Network Station is essentially a PC desktop minus slots, storage
options (such as CD-ROM and hard disk), multiple parallel and serial
connectors, a large power supply, and a cooling fan — in fact, the base unit has
no moving parts. The resulting "appliance" is an intelligent PC desktop that
features a Power PC 403 microprocessor (with additional, highly-tuned Java
processors planned for the future); 8 MB of ROM (read-only memory); 8 MB
of RAM (random access memory) expandable to 64 MBs; 1 MB of video
memory; 2 SIMM Sockets; a 10BaseT Ethernet or Token Ring 4/16
interconnect; as well as support for VGA, SVGA or XGA monitors. IBM also
includes a 102-key, PC-style keyboard.
One of the most striking features of IBM’s Network Station is its footprint —
this computing device is the size of a ordinary textbook. Real estate is saved by
limiting the number of slots and ports within the Network Station. The resulting
cabinet measures 190.5mm (7.5 inches) tall, by 266.7 mm (10.5 inches) deep,
by 31.75mm (1.25 inches) wide.
Also worthy of note, the IBM Network Station is a real power-miser. It draws
only 7 watts (watts — like in less than a tenth the power of your smaller
desktop light bulb) on an ongoing basis — substantially less than most PCs.
And due to this reduced power consumption rate, Aberdeen expects IS energy
bills to drop with the use of Network Stations — encouraging users to leave
their desktops powered-on overnight. This facilitates instant-on as workers
arrive at their Network Stations, and enables software updates to be performed
overnight.
The IBM Network Station — Operating Environment
Network computer operating environments are designed to be small and
efﬁcient — with optimized services for memory management, task
management, and network communications. These operating environments do
not need to deal with the complexities of identifying and managing multiple
add-on controllers, various ﬂoppy, hard and CD-ROM disk components, and
the like — the design focus is based upon accessing network applications and
displaying data, while mapping memory and other functions to gain system
efﬁciencies. To drive its Network Station, IBM uses a reliable and proven
Unix-kernel operating environment provided by Network Computing Devices,
Inc., Mountain View, CA.
The Network Station — Communications Subsystem
By their very nature, network computers must have optimized communication
protocol support for fast and efﬁcient communications, and ﬂexible
connectivity options such that IS managers can take advantage of existing
wiring plants as they look to deploy NC devices throughout the enterprise. The
Network Station uses TCP/IP as its basic communication protocol, and
currently provides connection to 10baseT Ethernet and 4/16 MB Token Ring
Local Area Networks (LANs).
Worthy of note, IBM will expand connectivity options in 1998 to include
coaxial and twinaxial cable support — enabling IS managers to take advantage
of existing 3270 and 5250 display terminal wiring plants for connection of NC
devices to various server environments. Aberdeen is aware of no other NC
supplier actively or aggressively developing coaxial and twinaxial cable
connections.
Terminal Emulators/Web Browsers
High-quality, feature-rich emulators and browsers are of vital importance when
accessing a variety of server environments. Failure to comprehensibly support
multiple display types can mean that applications may not be displayed
properly when accessed.

IBM Network Station customers will be relieved to know that IBM — the
company that designed and built the original 3270 and 5250 displays — has
brought its emulator technology know-how to the IBM Network Station. IBM
has engineered its own 3270, 5250 and X-terminal emulators using tried-and-
proven technologies from its PC and display terminal product lines. And IBM
currently uses the Spyglass browser for Web access. Further, IBM is in the
process of architecting its own IBM Network Station-optimized Web Browser;
and is also working with NetScape to natively implement Navigator 3.0 within
the IBM Network Station platform. Until these browsers are made available
(expected in Q2, 1997), the IBM Network Station will support the use of any
browser that now runs on a PC or RS/6000.
Application Environments
Many IS managers are astounded when they see the breadth of applications that
can be run by so-called "thin client" IBM Network Stations. Network Stations
can access:
Multiple terminal emulators — and can thereby access thousand of
existing applications on mainframe, Unix, AS/400, OS/2 Warp, NT and
other servers;
Existing Client/Server applications;
Java-based "applets" — enabling IBM Network Station users to run new,
fast-evolving Java-based applications on their desktops;
A wealth of personal productivity tools from a variety of suppliers — such
as Lotus Notes (through a Web Browser or via a Domino-based server), or
Microsoft NT-based applications in a timeshare-like mode.
From an applications perspective, IBM believes that IS managers and end users
will be delighted when they evaluate the performance characteristics of IBM
Network Stations running the market’s leading PC client/server and personal
productivity tools — in particular the Microsoft Ofﬁce product suite. IBM’s
Network Computer Division takes great delight in showing Network Stations
accessing an NT-based backend Microsoft Ofﬁce application server running
WinCenter Pro from Network Computing Devices to operate server-based
Word, Excel, and Powerpoint programs — and getting better overall
performance than users would get running those applications native on their PC
desktops! IBM stresses the theme that with IBM Network Stations, users have
the power of a server at their ﬁngertips.
Why the performance advantage? IBM attributes these performance gains to
the nature of the server-network computing architecture. The IBM Network
Station is essentially a fast, intelligent display device — a window-on-the-
world. Back-end servers often host the lion’s share of the application load —
and these back-end servers typically have 128 MB of memory that allow faster
access to applications and data than the 16 MB of memory found on most PCs;
extremely powerful microprocessors (usually more powerful than found on a
PC desktop); a high-performance Disk Access Storage Device (DASD); and a
powerful and optimized commercial-grade operating environment. The end-
user gets the advantages of two processors (one in the server and one in the
IBM Network Station) working on the task and lots more memory. IBM
believes that users of this architecture will often see better overall application
performance due to the fact that the back-end application environment is more
powerful than most desktop environments (especially 386, 486, and early
Pentium environments). Although some applications will not perform better
using an NC architecture, many IS managers are encouraged by IBM’s initial
Network Station performance testing — IS organizations may be able to
support more users with fewer servers.
From a Java perspective, Aberdeen has seen a marked increase in Independent
Software Vendor (ISV) commitments to Java-based development — a trend
that indicates that Java will be broadly accepted as a primary application
development environment for future client/server and network computing
applications. IBM has committed heavily to Java support and has ensured that
the Network Station can be ideally positioned as a Java desktop.
Doing the Math — The PC versus NC Cost Discussion
From a Network Station acquisition cost perspective, Aberdeen notes the
following:
IBM’s base unit list price for its Network Station, including keyboard,
quantity 1, is $695. A typical commercial PC desktop — with the
additional memory, additional disk, and the faster processors often
required to run NT — often exceeds $2,000. From the outset, IBM’s
Network Station cost less-than-half the purchase price of a typical
commercial PC desktop.
While results may vary by individual situation, Aberdeen estimates that
the appropriate apples-to-apples functional comparison of a networked
workgroup with PCs and a server — versus one with Network Stations
and a larger server — will show that the Network Station solution is
considerably less expensive on a per user basis.

But more important than initial acquisition costs are on-going support costs.
Administrators and managers are well aware of the costs and complexities
involved in managing large populations of distributed PC users — software
distribution, revision control, backup/restore activities, and asset control are but
a few of the problems IS personnel face when dealing with diverse PC
environments. Further, PC users have a tendency to customize their own
desktops — adding unauthorized software or tweaking the systems
conﬁguration — often requiring technical assistance to repair the damage. In a
world where many IS professionals claim that it costs between $4,000 and
$15,000 per year to support each networked PC user, Aberdeen notes that the
IBM Network Station has no moving parts, and that its operating environment
requires minimal service. A network computing architecture can profoundly
effect the cost-to-manage a distributed desktop environment.
IS managers should be aware that IBM’s Network Stations can be administered
centrally using IBM’s Network Station Manager — a Java and HTML-based
Internet management tool that can ensure that users get a high level of desktop
service and remain in compliance with corporate desktop standards. With this
tool, IS administrators can conﬁgure user desktops, establish application and
server authorization rights, manage desktop display options, and provide users
with the information access services they need — while also preventing users
from modifying or breaking their desktops and thereby increasing IS service
costs.
Many IS managers and administrators believe that the deployment of NC
devices will actually lead to a higher level of desktop service for their end-user
communities at far less cost — applications can be better managed and more
easily installed when located on a centralized server environment.
Finally, a key cost issue is upgrades. The useful life for a PC with its native
operating environment and applications seems to be approximately 3 years.
And then it is typically replacement time — out with the old, in with the new
in-box upgrading is simply too costly. With IBM’s Network Station, the
desktop device should have a useful life of approximately 5 years during which
all operating environment and application upgrades can be made at the server at
one time. Longer life — lower upgrade costs — is clearly a value proposition
that ﬁnancial management will like.
But It’s From IBM — Is The IBM Network Station Really
Open?
IBM, Oracle, Sun Microsystems, Netscape, and Apple announced a common
network computing standard on May 20, 1996 — the NC Reference Proﬁle 1.
This reference proﬁle serves as a "standard" basis and has already been
endorsed by numerous other suppliers — although IBM, Sun, and Oracle have
added extensions to the products based upon the Reference Proﬁle to meet their
customers’ speciﬁc needs for an optimized network computer for their speciﬁc
environments. These modiﬁcations are appropriate — the Reference Proﬁle is
simply a functional speciﬁcation. Suppliers can and should take numerous
different approaches to how they architect and build their products to meet their
speciﬁc customers needs.
Since the reference proﬁle was released, IBM has delivered its own product —
the IBM Network Station. Oracle has been encouraging partners to develop
products according to its speciﬁcations and several such products are now
available. And Sun has introduced its JavaStation line.
With this much hardware supplier support, the independent software vendor
community has followed with its support also. This application support is the
basis of why the IBM Network Station is neither proprietary nor represents
dead-end technology. IBM, its Reference Proﬁle alliance partners, and the
independent software community are all achieving the base level standards
compliance that will ensure openness. Customers with legacy 3270 and 5250
applications can keep their applications while moving into the new, open world
of the Internet with IBM’s Network Station on their desktop.
Summary Observations
With IBM’s Network Station network computer, information workers and IS
man-agers now have a new, far less-costly option to evaluate for desktop
computing. IBM Network Stations can perform many of the same information
gathering and data manipulation tasks that full-ﬂedged PCs can perform — but
cost only half as much as a PC platform to purchase and manage. This value
proposition — the ability to perform PC information gathering tasks; run PC
client/server and personal productivity applications; while offering greater
manageability at a reduced platform cost — can be expected to appeal strongly
to price-sensitive user organizations, as well as overburdened systems
managers and administrators looking to reduce desktop management costs.
Aberdeen expects that the deployment of Network Stations will lead to
reductions in overall desktop computing costs while improving the level of
service that the IS organization can provide.
Information workers who tested IBM’s network computing concept were
surpris-ed by the simplicity and effectiveness of IBM’s Network Station. The
Network Station provides straightforward and transparent access to applications
and data located from within and outside of organizational boundaries — users

need only point-and-click between varied server environments to access the
information or applications they needed — and users were allowed access
provided they had the proper access privileges. Further, many users
experienced improved application performance because the bulk of their
client/server and personal productivity applications executed on powerful
servers rather than on their less-powerful PC desktop platform.
IS managers have been delighted with the IBM Network Station — due
primarily to the ability to better manage desktop resources and services from a
central point of control. By centralizing management activities at the server
level, IS managers have found that management services such as software
distribution, backup/restore, and other redundant activities can be more easily
and efﬁciently handled — resulting in better service to large desktop user
populations.
But one IS organization — the help desk — is probably the group that is most
appreciative of IBM’s Network Station technology. Due to the lack of local
disk drives, the IBM Network Station stops users from tampering with their
desktop conﬁguration settings and prevents users from loading rogue,
maverick, and often unauthorized PC software programs — and sometimes
viruses — that can result in hours of unnecessary PC desktop troubleshooting.
Legions of information workers are still insisting that they require a personal
computer to access enterprise/departmental/workgroup data, surf the Internet,
and run various client/server and personal productivity applications. But, with
the introduction of the IBM Network Station, users who ﬁt this proﬁle now
have a new, less-costly option for desktop computing. Aberdeen encourages IS
managers to assist end-user organizations in the reevaluation of their desktop
technology requirements — and, where appropriate, to counsel their end-users
to evaluate IBM’s Network Station. With the arrival of IBM’s Network Station,
end-user organizations are about to ﬁnd that there are many desktop
deployment situations wherein a PC desktop platform simply represents
processing overkill.
