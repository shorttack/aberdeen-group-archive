## Abstract

Aberdeen Group examines the forthcoming Informix MetaCube 4.0 release, a Relational OLAP environment that leverages Informix's Dynamic Scalable Architecture (DSA). The profile argues that emerging customer-centric applications will demand more from data warehouses and data marts, requiring RDBMS-native ROLAP tools that can scale user populations, adapt to changing data/business models, and integrate with intranet strategies. MetaCube 4.0 is positioned as solving these challenges by intelligently sharing critical tasks between the RDBMS and the OLAP layer.

## Key Findings

- Customer-centric applications will demand more from data warehouses/marts (Aberdeen)
- ROLAP must be RDBMS-native to deliver speed and scalability for large-scale DSS
- MetaCube 4.0 leverages Informix DSA by sharing tasks between RDBMS and OLAP layer
- "Surround strategy" combining DSS with RDBMS administration cited as critical
- Intranet adoption cited as key driver increasing requirements for adaptive ROLAP

## Context & Prescience

MetaCube was acquired when IBM bought Informix's database business in 2001. The product was eventually discontinued. However, Aberdeen's core prediction — that RDBMS-native OLAP would become essential and that intranet/internet strategies would drive DSS adoption — proved accurate. The market ultimately consolidated around embedded BI in RDBMS platforms (IBM Cognos, Oracle OBIEE, Microsoft SQL Server Analysis Services) and cloud BI (Tableau, Looker, Power BI).
