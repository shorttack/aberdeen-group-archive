# aberdeen-1996-3com-reconciling-clientserver-development-internet

**Title:** Reconciling Client/Server Development and the Internet  
**Author:** Aberdeen Group (Tim Sloane)  
**Date:** November 1996  
**Type:** impact-report  
**Sponsor:** 3Com Corporation  
**License:** CC-BY-4.0  

## Summary

Aberdeen Group impact report advising IS professionals how to reconcile existing client/server investments with the need for Internet-based deployment. Authored by analyst Tim Sloane, the report argues for Web-enabled client/server tools over dual-tool strategies or Java-only approaches, noting Java's 1996 limitations for complex OLTP. Predicts EDI will move to the Internet within 5 years and identifies the move from CGI to application-server architectures.

## Assessment

| Dimension | Rating |
|-----------|--------|
| Importance | medium |
| Relevance | high |
| Prescience | high |

Core predictions confirmed: EDI over Internet (AS2 by 2002), app-server replacing CGI, browser as server-centric 'client for the masses'. Java limitation prediction accurate for the era.
