### Executive Summary

Client-server technologies, and the advanced business solutions that exploit them, are today proving to be as much the catalyst for business change as the vehicle by which business change is implemented. Aberdeen has asserted that those suppliers of Client-Server Solutions (CSS) that would move beyond their line-of-business functionality to incorporate significant value-added technology and business capabilities -- advanced state-of-the-art graphical user interfaces (GUIs), workflow, integrated e-mail, object-oriented operability and explicit support for constant business process improvement -- would become the market leaders. These assertions are proving true.

Lawson Software, with considerable expertise in open systems and client-server technologies, continues to eclipse its traditional midrange AS/400-centric competitors. Moreover, the company seriously challenges current CSS market leaders with an integrated graphical desktop that provides for a multi-layered implementation of business process management and improvement that stands unique among CSS.

### The Enterprise-capable PC Desktop

Windows-based GUI desktops are a far cry from the IBM 3270 and 5250 character terminals that the majority of business applications still running today were designed to utilize. The GUI at its best provides an intuitive and accessible means through which the end user -- from executive suite to shop floor -- interacts with the automated systems and processes required to accomplish the job at hand. At its worst, a GUI can be a nightmare combination of illogical, confusing and disconnected desktop and corporate applications that inhibit individual and business productivity.

The most recognizable GUI perspectives and resulting products are:

- **Windows Program Manager-biased** — Oriented towards basic office functionality; applications operate singularly while offering data- or file-level integration (DDE, OLE)
- **Electronic Mail-biased** — Primary desktop function is deemed to be Email; access to non-Email applications is accomplished by task switching
- **Groupware-biased** — Designed to provide a collaborative workgroup environment for non-transactional, non-repetitive activities
- **Application-biased** — The business application "takes over" the desktop; usually needs to be exited to engage related applications

Aberdeen asserts that the best business-desktop perspective understands the operational context and best integrates the multiple technologies capable of supporting the intended business processes.

### An Open Enterprise Desktop

Lawson Software, with over twenty years of core-business applications experience, has learned the importance of the business perspective when it comes to developing high-impact, high-productivity software solutions. This experience is nowhere more evident than in the company's Open Enterprise Desktop (OED) technology.

OED is developed in C and C++, making use of Microsoft's Win32 Application Programming Interfaces (APIs), resulting in a product compliant with the specifications for the Windows 95 and Windows NT 32-bit multi-tasking operating environments. OED and the Lawson Enterprise CSS are fully Email enabled. The desktop provides iconic activation and switching to any MAPI-compliant mail system. VIM support follows with Version 7.0.

Lawson has raised the bar by using the mail transport to carry embedded active and passive application objects. These application objects are represented as an icon within the body of the Email message. This capability allows the Lawson CSS user to mail an application object (e.g., journal entry, purchase requisition, or inventory status) to any other user reachable by Email.

Aberdeen understands that Lawson's R&D group is already at work implementing the next release of OED that will optionally allow for Internet application deployment and execution capabilities based on SUN's Java technology.

### Toolbars, Tabs and a Desktop with Personality

Lawson has developed the Lawson Toolbar, which provides a business application context across the virtual desktop. This desktop business perspective functions across all applications -- from Lawson, Microsoft and even other non-Lawson CSS.

Lawson's Intelligent Folder Tabs use the file folder metaphor of desktop organization. By selecting an application button the user can essentially hot key to specific sub-tasks or application functions without having to navigate out of the folder or transfer via the Lawson Toolbar primary application icon.

### Enterprise Workflow

In the manufacture of a product, as well as in the delivery of a service, there are normally multiple discrete steps or activities that should be performed in some logical sequence, progressing from activity inception to completion. This sequence is referred to as a business process.

The implementation and control of the progression between the steps of a process is referred to as workflow. Workflow automation is the means by which a business process is implemented through system, communication and software technology.

### Aberdeen's Key Evaluation Criteria for Integrated-workflow CSS

Aberdeen evaluates CSS integrated workflow automation technologies based on three main criteria:

1. **Flexibility** — Ability to support cross-domain functionality spanning multiple organizations and systems
2. **Scalability** — Ability to deploy flexibly matching diverse computing infrastructures
3. **Productivity** — Capture and report key process-effectiveness data points for ongoing improvement

### Lawson Workflow

Lawson Software has been working toward an open systems best-of-breed integrated workflow implementation for its Lawson Enterprise CSS, developed under the code name Edinburgh, for over two years.

Lawson's workflow architecture is designed for client-server deployments using flexible message-based communications within an object-oriented framework. Workflow processes are managed and monitored via the Lawson Universe Repository and active Process Agents.

Flexibility is achieved through a plug-and-play approach using standard APIs and server-to-server transaction interfaces. Cross domain workflow spanning Lawson and non-Lawson applications can be supported using industry leading technologies such as Wang's OpenWorkflow or Filenet's Visual Workflow.

Lawson customers can choose to implement workflow at the:

- **Personal level** via the advanced organizational and navigational capabilities of Lawson's OED
- **Departmental/Workgroup level** via OED and the Lawson Departmental Workflow Engine
- **Enterprise level** with OED and Lawson's Process Suites

The company is delivering **four key business Process Suites** incorporating **20 predefined best practice process models** such as AP Invoice Approval, Requisitions Approval, Credit Check and Employee Review. The first available Process Suites are:

- **The Procurement Suite**: General Ledger, Accounts Payable, Purchase Order and Requisition
- **The Supply Chain Suite**: Inventory Control, Purchase Order, Sales Order Entry, Sales Analysis, Billing, Accounts Receivable, Accounts Payable, General Ledger
- **The Human Resource (HRMS) Suite**: Personnel, Flexible Benefits, Payroll and KIOSK

Aberdeen calls the iterative improvement methodology **Rapid Process Development (RPD)** and cites Lawson as in the forefront of bringing this capability to the market.

### Maximizing the Benefits

Aberdeen advises IS buyers: never use workflow to automate manual processes that already do not work or do not meet quality or process requirements. Reengineer the process first and then incorporate it into the workflow-enabled environment.

IS re-education is a must. IS business analysts are finding their contribution expanding beyond the traditional role as technologists and are becoming more fully involved in the direct business solution implementation.

### The Competitive Landscape

Enterprise-class CSS suppliers have all stated intentions to deploy workflow capabilities. Among these suppliers are Oracle, PeopleSoft, and SAP AG.

**Oracle**: First efforts with workflow integration via Cooperative Application Initiative (CAI) program launched mid-1994, in partnership with Recognition International (now BancTec). Oracle Workflow announced for initial availability Q1 1996.

**PeopleSoft**: Version 5.0 implements proprietary workflow via three means: Email-based (messaging), database-based (stored procedures and triggers), and planned APIs for third-party workflow interoperability.

**SAP AG**: Entered workflow in spring 1995 with third-party Cascade (BancTec/ICS Deloitte). Since announced its own internally-developed workflow with phased delivery in R/3.

Aberdeen finds that these suppliers, including Lawson, have targeted and can meet -- or intend to be capable of meeting -- the Enterprise Category of workflow functionality.

### Aberdeen Conclusions

Lawson Software with its one-two-punch of Open Enterprise Desktop and integrated Lawson Workflow is delivering a highly desirable and highly competitive value-added CSS combination.

The Lawson products and technology discussed herein are also available and supported on the AS/400 platform operating with Version 3 Release 6 of the OS/400 operating environment. The OED and Lawson Workflow combination is unquestionably a unique and compelling offering in that market, and potentially a killer app versus the company's traditional competitors.

Lawson Software now can offer: a recognized industry leading CSS for financials, materials management and human resources; an enterprise-class client desktop implementation; a floor to ceiling multi-level open systems workflow architecture; and demonstrated capacity to deliver against the key evaluation criteria of flexibility, scalability and productivity.
