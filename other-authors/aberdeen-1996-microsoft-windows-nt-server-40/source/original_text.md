### Market Position

Windows NT Server 4.0 -- which has become the most talked about network operating system -- is the latest release of Microsoft's multitasking, multithreaded networking product. Starting with version 3.51, NT Server began reaching a level of maturity that has commanded attention from the trade press, and is garnering editorial awards and consultant recommendations. Even more importantly, NT Server is now receiving serious consideration on the part of IS and independent software vendors (ISVs) as the application server of choice.

Its main competitors are **Novell NetWare**, the various flavors of Unix, and **IBM's OS/2 Warp Server**. Unlike Microsoft's other operating systems, NT Server is not solely identified with a specific hardware architecture. There are versions for Intel-based servers, as well as most RISC machines -- including several suppliers using the MIPS architecture, Digital's Alpha, and the PowerPC systems from IBM, Motorola, Groupe Bull and others.

No longer the stepchild of networking, Windows NT Server 4.0 now has strong capabilities which span file and printer sharing with use as an application server, and add key Internet/intranet capabilities.

NT 4.0 is tuned for **four-way symmetric multiprocessing (SMP)** -- which allows for inexpensive scalability -- and several system suppliers are working on NT Server clustering for both high-availability and increased scalability. Within a year of its introduction, Aberdeen expects NT 4.0 to be seen as a heavy workhorse as a database and application server at the workgroup and departmental level.

With new directory directions already announced for Windows NT Server 5.0, Microsoft may have finally created a practical and compelling network offering.

### Inside the new NT Server

According to Microsoft, focus groups identified the following reasons why users chose NT:

1. Ease of set-up, deployment, management, and use
2. Compatibility with existing systems
3. Ability to use the same operating system for application servers, file servers, and communication servers
4. Suitability as an Internet platform because of the built-in Web server

Reflecting these selling benefits are **four major themes** in the changes to NT Server:

1. Convenience and compatibility
2. Integrated Internet/intranet capabilities
3. Network integration
4. Performance enhancements

---

### Convenience and Compatibility

NT Server now sports the well-received **Windows 95 user interface** rather than the previous Windows 3.x look and feel. Features such as the task bar and right mouse button pop-up option menus are particularly helpful in network administration.

New wizards for administrative functions allow less-experienced personnel to perform many network administrative tasks -- adding users, user group management, adding printers, managing permissions, installing modems, tracking software licenses -- without in-depth knowledge.

The Windows NT Directory Service (NTDS) can be replicated to multiple locations for back-up and load balancing. Users can access the entire network with a single login. The domain approach supports **up to 40,000 users in a single domain** and **up to 500 trusted domains**.

A beta version of NT Server 5.0 is supposed to ship by the end of 1996, but this leaves NT Server 4.0 lacking a critical area: LDAP-based directory for heterogeneous networks. **LDAP (Lightweight Directory Access Protocol)**, developed at the University of Michigan as an X.500 directory client, was adopted as a standard by major networking vendors earlier this year.

---

### Integrated Internet/Intranet Capabilities

The **Internet Information Server 2.0 (IIS)** is now an integral part of the network operating system. Microsoft claims this second generation product is **40% faster than the 1.0 version**. IIS supports Web service, gophers, FTP, ODBC database connectivity, and administrative services. Administration is possible using any Web browser.

**PPTP (Point to Point Tunneling Protocol)** is a new mechanism that provides a secure method of passing network packets over a public network by encapsulating, and even encrypting, the packets. This allows the creation of secure private intranets using public data networks.

---

### Network Integration

**Distributed COM (DCOM)** allows software components to communicate across networks, including the Internet. Since it uses the same tools and technologies as COM, DCOM does not require extensive retraining and reinvestments.

NT Server 4.0 provides Multi-Protocol Routing support for routing over multiple protocols, including IPX/SPX, TCP/IP, and AppleTalk.

### Performance Enhancements

**NT Server 4.0 vs. 3.51 file/printer sharing performance improvements on 100MB network** (Source: Microsoft Corporation, July 1996):

| Metric | Improvement |
|--------|-------------|
| Network frames transmitted | -32% |
| Bytes written to disk | -27% |
| Interrupts | -50% |
| Server code path length | -33% |

**Application server performance improvements:**
- ~30% reduction in resource contention
- ~35% fewer resource allocations
- Pentium and PentiumPro optimizations for Intel-based servers
- New MP (multi-processor) communication protocol

---

### Application Servers

The majority of installed networks are used for traditional file and print services. However, the dynamic growth in servers is occurring with those used as application and database servers. And it is in this market segment that NT Server has flourished, and will continue to do so.

There may be a better product currently available and products with a significantly larger installed base, but **the long-term market winner will be in large part determined by the number of ISVs creating applications for its operating system.**

With this in mind, Microsoft began its initial roll-out of Windows NT Server with a software developers' conference in 1992 -- which was attended by **more than 5,000 ISVs** -- and shipped **nearly 60,000 software developers' toolkits** by the end of that year.

---

### Conclusions

Windows NT Server 4.0's emphasis on distributed computing and on very high level tools, including wizards, for easy installation and configuration will play well in small companies, work groups, and departments needing application and database servers.

**Aberdeen has not yet found a company or agency which is not in some stage of implementing an NT Server-based application.** All of the interviews conducted over the past year with those responsible for making buy decisions indicate that installations of NT Server-based applications will show a **major increase in 1997**.

In some ways, Microsoft has become **the IBM of the 1990s** -- a "safe" bet to buy because of its marketing clout, its ability to attract key ISVs -- and its persistence with improving its core products.
