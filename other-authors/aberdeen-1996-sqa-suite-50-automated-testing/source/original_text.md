SQA Suite 5.0: Innovative, Integrated, Automated Testing Tools
Preface
No developer can afford to overlook the need for automated application testing tools that are now a crucial requirement in the development lifecycle of increasingly sophisticated and
complex client-server applications. While certain aspects of development are made easier by the dominance of Windows as a desktop client, the developer productivity and shortened time-
to-market beneﬁts of component-based development, and the prevalence of highly effective visual programming tools, ensuring the quality of a ﬁnished product is even more difﬁcult.
Adding to this dilemma is the dynamic nature of the client-server application development process that is characterized by continuous changes requiring ﬂexibility of design and delivery,
ubiquitous deployment, and frequent redeployment. In other words, the ﬁnished product becomes a moving target.
Polished and bug-free applications, shortened QA cycles, more maintainable code and timely delivery are the results of continuous automated testing of Windows client-server applications.
Aberdeen believes that any technology buyer worth his salt will seriously consider some form of automated testing tools within the next two years. Testing tool buyers will be looking for the
ability to test Object Linking and Embedding (OLE)-enabled components, multiple versions of Windows (Windows 3.1, Windows NT, and Windows 95), and to leverage testing scripts in
both 16- and 32-bit without modiﬁcation.
This Proﬁle outlines SQA, Inc., and its innovative SQA Suite 5.0 of Window automated testing tools that excel at meeting these demands.
Executive Summary
SQA Suite 5.0 is comprised of four products that deliver Windows automated client-server end to end testing. Each of the tools works effectively on its own, and combined, they produce a
greater impact. This integration, along with an aggressive product enhancement schedule, makes SQA a serious contender in the testing tool arena. SQA Suite is based on an integrated
repository that can be used to produce highly informative reports on test data, store test scripts for future use, deliver defect tracking and management, and handle workﬂow rules and
security.
SQA Process and SQA Manager deliver a methodology and tools that provide Process Management capabilities to aid in the planning, design, reporting, and analysis of the test cycle, with an
additional defect management module. The tracking, management and resolution of defects within the developing application is a substantial beneﬁt of SQA’s methodical approach to
building healthy software. Defects can be assigned priorities—the higher the priority, the greater the problem—which are then eliminated through the development cycle. At each stage
developers can be sure of exactly how many bugs remain to be tackled and which are more threatening than others. Developers can then determine criteria for release of the application from
the testing cycle. SQA Process presents a blueprint for implementing proven automated testing solutions that are repeatable, measurable, and consistent across testing projects.
SQA Robot, and SQA LoadTest fulﬁll the further requirements of client-server testing by delivering tools to create scripts, and execute tests. On the client side SQA Suite supplies a unique
ability to test OLE/OCX, ActiveX and GUI objects used to create applications, a crucial advantage when testing applications in today’s multihued Windows environment. SQA supports
server-side load and stress, performance, and multiuser testing via SQA Robot and SQA LoadTest, which generate scripts that are then executed against the server for assurance that
applications will scale and perform up to par.
A unique feature of SQA Suite 5.0 is its underlying server-based repository that integrates the tools. Integration at this low level allows the tools to share test scripts, share test data, and
provides a united product for end-to-end client-server application testing. The test repository is the crux of SQA’s integrated testing methodology. Through it SQA can deliver historical
analysis of test data, management of defect tracking through each stage of testing, and management of the types and frequency of tests needed for thorough testing.
Aberdeen believes that testing should be an integral part of the application development cycle and that, without continuous testing and defect management, applications will be more costly to
develop, delayed longer in the development phase, and delivered with more bugs as client-server applications increase in sophistication and delivery. This is esecially true when testing more
complex 32-bit applications. SQA Suite 5.0 has an integrated testing suite designed speciﬁcally for testing Windows client-server applications and for alleviating the issues associated with
cross-Windows 16-bit and 32-bit application testing.
SQA Suite 5.0 delivers signiﬁcant beneﬁts with a full suite of tools that support Windows NT and Windows 95 automated application testing. As existing applications are migrated to new
Windows platforms, as new versions of client-server application development environments support 32-bit development, and as development efforts are moved to the robust Windows NT
platform, development staffs will require additional testing support and robust 32-bit testing tools. SQA Suite 5.0 helps ease the migration issues associated with these transistions and
furnishes an integrated testing toolset that supports all Windows platforms, delivering a distinctive advantage to 32-bit developers.
Aberdeen strongly suggests that IS managers serious about delivering robust client-server applications under Windows evaluate this tool suite.
Market Position
Windows development has signiﬁcantly changed the testing tools market. Traditionally testing was an everyday part of the development cycle of heavily designed applications that typically
took two to three years to build. As 4GL graphical tools made building client-server applications easier and quicker, testing often fell by the wayside. Today developers and IS management
are faced with shortened (less than a year) delivery deadlines, demands for highly sophisticated feature-rich applications, and a lower tolerance for imperfect software. As a result, they are
trying to reintegrate testing methodology into the development cycle. This is a proﬁle of SQA’s typical customer.
SQA differentiates itself in the testing tools market by its ﬁrm stance in the Windows automated testing arena, rather than a cross-platform or Unix-based approach. This focus on Windows
client-server has enabled SQA to focus on the difﬁculties of testing applications that will be deployed on various versions of windows, including 16- and 32-bit versions. Thus SQA has made
a name for itself both by the frequency with which adds new functionality to its software and by the innovative way in which it tackles Windows testing. A leader in providing 32-bit
Windows application testing, SQA also prides itself on its ability to test components such as OLE-enabled OCX objects and the integrated nature of its testing tools via a repository.

Aberdeen Client-server Testing Criteria
Aberdeen research has shown that users require certain basic automated testing features such as:
• Robustness—delivering a tool that stands up to the rigorous trials associated with very immature code, giving developers the ability to gather the most information possible throughout
development;
• Scalability—support for testing very large applications as well as load testing capabilities for testing the capacity of an application for growth;
• Flexibility—the ability to handle changes in the test scripts during execution, leverage test scripts across multiple versions of Windows, and carry out component testing;
• Breadth of Features—support for a wide range of testing scenarios, including unit, performance and load testing;
• Administrative Support—support for the design and management of the testing process;
• Ease-of-Use—providing a graphical user interface and automated features to help developers be more productive in the test cycle;
• Integration—having a close integration strategy within the testing toolset as well as with major popular third-party development toolsets and design tools, and;
• Strategic Testing and Implementation Support—delivering support for the design, creation, and implementation of an effective test methodology and test suite.
Aberdeen believes that Windows automated testing tools that meet IS criteria will be instrumental in radically shortening development time, possibly by as much as 50 percent in the case of
complex client-server applications and especially with transaction heavy systems.
Robustness
One of the difﬁculties faced by developers during the continuous testing process is keeping the test tool up and running while testing unstable code during the ﬁrst stages of the process. A
testing tool must provide a robust and stable environment in which the most amount of information can be gathered and thus used to improve code for the next round of testing. SQA Suite’s
test repository is based on an industrial strength database server that delivers the speed and data integrity features needed to gather test data throughout the text cycle. SQA Manager provides
a complete set of test management tools and can automatically send electronic mail alerts to notify developers of changes or defects in the application code.
SQA’s Object-Oriented Recording, a technique that automatically adjusts to changes in application objects and windows, combined with SQA Suite’s SQABasic scripting language delivers
the ease of GUI script creation with the power of a programming language.
Scalability
The ability to handle the testing of large applications, many testers, and an increased test load are inherent to SQA Suite because of its test repository. The ﬁrst and only Windows automated
testing toolset with a server-based repository, SQA Suite can use Microsoft Access, a virtually maintenance-free database that can run on almost any platform, or Sybase SQL Anywhere, an
RDBMS that provides SQL and cross-platform portability, as well as a small footprint and links to the highly scalable Sybase SQL Server RDBMS. Thus, this scalable repository can handle
the testing of a large application or a large number of QA testers. Every copy of SQA Suite includes a runtime version of SQL Anywhere and Microsoft Access.
Flexibility
The underlying test repository also allows the developer to test an application on multiple versions of Windows (Windows 3.1, Windows NT, and Windows 95) without having to completely
rewrite test scripts. SQA has pioneered a technology called Smart Matching that adjusts both 16- and 32-bit test scripts to handle the differences between 16- and 32-bit versions of Windows.
This ensures that test scripts can be leveraged across Windows platforms. For example, Smart Matching will automatically adjust for differences in the way that Windows 3.1, Window NT
and Windows 95 display window captions.
Breadth of Features
An integrated suite of testing tools delivers the greatest impact through support for a full range of application tests. The most effective test suite will incorporate such test types as:
• Load Testing—simulates users requesting services from the server and pinpoints application bottlenecks;
• Stress Testing—mimics maximum load on an application for the longest possible timeframe to determine utmost server capacity;
• Conﬁguration Testing—determines the optimal combination of hardware and software to run the application most efﬁciently;
• Multiuser Testing—simulates multiple users running simultaneously;
• Performance Testing—pinpoints issues during various usage scenarios and helps in ﬁne-tuning applications for best performance;
• Unit Testing—breaks down the application into more easily testable individual modules;
• System Testing— links application units back together into full-ﬂedged application for testing on the exact platforms on which it will be deployed; and,
• Integration Testing—ensures that the application will integrate well with other applications in its environment.
SQA LoadTest calls and executes the tests, created by SQA Robot scripts, needed to meet users’ testing requirements in each of these categories. SQA’s testing suite will excel as an all-round
testing tool when SQA releases a product currently in development, an extension to LoadTest, that simulates multiple users hitting the database server simultaneously. Currently SQA
LoadTest requires a separate PC for each client process, rather than one PC simulating multiple clients. The addition of the new LoadTest feature will strengthen SQA’s product offering by
giving developers the option of both test scenarios and further cementing SQA Suite’s position as a thorough end-to-end testing tool. SQA LoadTest offers a graphical interface and
seamlessly integrated client-server testing with SQA Robot and SQA Manager.
In addition SQA LoadTest helps mitigate the snarls of getting up and running with automated testing tools via a single integrated installation and helps lower initial costs by not requiring
users to license SQA software to run on the agent PCs. SQA LoadTest runs on the server and is licensed according to the number of PC agents that will be connected to the server and run by
SQA LoadTest.

SQA LoadTest provides 17 built-in graphs in SQA LoadTest, allows developers to custom design their own speciﬁc graphs, automatically generates statistical reports of varying levels of
detail, and uses the test repository to store performance metrics.
Administrative Support
Because client-server and cross-Windows testing is a new area for many IS groups, it is necessary to provide strong administrative support throughout application testing. It is also crucial to
ensure that every piece of the application is tested that all results are logged, and that developers take possession of and effect resolution of detected bugs. Hundreds of test scripts must be
generated, executed and managed efﬁciently to preserve the beneﬁts of automated testing. Poorly executed testing is sometimes worse than no testing at all and will introduce more
difﬁculties into the testing process rather than eliminate them.
SQA Manager furnishes comprehensive test planning, workﬂow management and defect tracking to ensure that application testing runs smoothly and accurately. A graphical test planning
utility allows testers to match test suites to the predetermined application test requirements. Graphing and reporting are an integral part of SQA Manager. Several common graphs and reports
are included but can be easily customized to meet speciﬁc needs.
Tests can be planned using the tree structure of SQA Manager’s test planning module. Developers can track test assets, cross-reference test plans with speciﬁc test requirements, break large
applications into smaller more testable modules, and organize test procedures and results within SQA Manager. The more tightly the tools are integrated, the better the test management will
be and the more dramatic the results.
Strategic Testing and Implementation Support
Although automated testing is gaining importance in the development lifecycle, many potential users are not aware of the planning and effort it takes to effectively implement useful test
scenarios. In some cases testing is viewed as more of a burden than a boon because of resources needed to make it an effective piece of the development process. If improperly administered,
testing can prolong application development and miss important bugs. SQA addresses these IS concerns with SQA Process, an integral part of SQA Suite 5.0. SQA Process is a methodology
bolstered by utilities that support the concepts and techniques needed to test Windows applications.
SQA Process alone is one of the most compelling reasons to use SQA Suite 5.0. A unique offering in Windows automated testing, SQA Process adapts to testers of various levels of expertise,
both novice and advanced, allaying IS fears that additional QA stafﬁng will be required to handle automated testing. It provides a step-by-step process through which developers can come up
with a measurable testing process for applications, deﬁne a testing strategy that runs parallel to expected output, and addresses many types of automated tests, including performance, stress
volume, conﬁguration and installation testing.
To help manage the process according to the SQA Process blueprint, the methodology also helps to set responsibilities in the test process, estimate the scope of the project, come up with the
optimum test scenario for a particular project, and produce reports on test coverage and defect tracking.
SQA Process covers six main areas of the test cycle:
• Planning—a test plan ensures that all aspects of the application is covered and sets criteria for measuring and controlling the process;
• Design—deﬁne test procedures and test cases that verify the test plan criteria are being met;
• Development—development of the actual scripts that will execute the tests that were previously designed
• Execution—actual testing process, running the test scripts against the software;
• Evaluation—results are reviewed to determine if test criteria have been met and if the software can be deployed or must be corrected; and
• Tracking—management of test and defect resolution information via SQA’s integrated test repository.
Ease-of-Use
Users of automated testing software tend to be most productive when the interface to the tool is graphical, self-explanatory, and functions at a high level of abstraction. SQA Suite 5.0 excels
as high-productivity graphical testing tool.
For the times when developers need to take scripting to the next level, SQA Robot’s SQABasic scripting language is compatible with and will be familiar to Visual Basic users. A single
installation process for the entire SQA Suite eliminates the integration puzzles and tedious effort associated with melding together multiple individual third-party solutions for planning,
creating, executing, managing, and mapping the test process.
As mentioned above SQA Suite-created scripts are portable across 16- and 32-bit applications on Windows 3.x, Windows NT, and Windows 95, eliminating the need to create separate scripts
for an application that may be deployed in each environment.
Integration
As stated previously, to be of maximum effectiveness automated testing must take place continuously throughout the development cycle. Thus automated testing tools must be closely
integrated with major popular GUI development tools, such as Microsoft’s Visual Basic, Powersoft’s PowerBuilder, and Centura’s Centura. SQA Suite 5.0 has optimized its product to work
with each of these leading toolsets.
In addition, SQA Suite applies a new technology called Object Testing to the testing of these client-server applications. Object Testing enables SQA Suite to inspect and verify all the
properties of an object, whether the properties are visible or not. For example, a developer may test a SQL call sent by a Visual Basic OCX to a database.
SQA Suite for Visual Basic allows developers to test Windows applications on any Windows platform. It can test C and C++ code and integrates closely with Visual Basic at the object level,
which allows SQA Suite to test OCX, Internet, ActiveX, and VBX controls. SQA Suite for Centura and SQA Suite for SQL Windows supports Table Windows, a feature unique to Centura
products. SQA Suite for PowerBuilder provides comprehensive object-level testing through PowerBuilder’s Testing Application Programming Interface (TAPI), including the PowerBuilder
OCX and ActiveX objects.
Another reason that SQA Suite is able to so smoothly and tightly mesh its testing tools with each of these client-server application development environments is its ability to record objects
by the unique names given to the object by the programmer. This contrasts greatly with the older method for tracking Windows GUI elements that called for developers to map the on-screen
elements. However, if changes were made to the location of an element or its name on-screen were changed, the test would fail. This can result in a great deal of tedious upkeep during the
development of an application and is not practical when applied to continuous testing.
Instead of this mapping method, SQA Suite supports Object-oriented Recording, the use of unique object names when invoking graphical screen elements in a test scenario. This ensures that
no matter where the object may reside on the screen, the test script will call it correctly and execute the test properly. Thus developers may move the GUI elements around and not be

concerned with mapping every change.
Because SQA Manager is tightly integrated with SQA Robot, test scripts and execution data are automatically updated within SQA Manager.
Using SQA Suite Effectively
Even ignoring the basic truths about automated testing,—that fewer testers can clean up more code and unattended processes can run around the clock for better results—SQA Suite can
deliver beneﬁts that recur throughout the application’s lifecycle. Continuous testing during development can save time and money by targeting problems before they escalate and cause even
more bugs later on in the development process. Development times are shortened by having cleaner code earlier on in the process and fewer bug-chasing details delay delivery. Robust and
relatively bug-free code, or well-documented problem issues, are easier to address in subsequent releases and well-tested code is easier to modify and maintain over time.
Aberdeen ﬁnds that buyers implementing Windows automated testing tools should apply the following principles:
• Make a long term and high-level commitment to automated testing;;
• Follow a methodology that is designed for handling automated testing;
• Start the testing process early and continuously test code until the ﬁnal version;
• Keep track of all reported bugs and prioritize them;
• Establish testing goals at the outset and stick to them throughout the development cycle, measuring the project’s progress;
• Keep historical data on the testing process, comparing results over the lifespan of application development; and,
• Test custom components before distributed for multideveloper use.
These basic rules, when applied to automated testing, will most likely boost development productivity, decrease time to delivery, and ease maintenance cycles, especially when combined
with SQA’s integrated testing tool suite.
Internet Direction
SQA will deliver Internet/Intranet -enabled versions of its testing tools that will suply test management, defect tracking, and functional, component, object, load, and stress testing
capabilities. The new tools will allow developers to test Java, and Internet ActiveX controls; provide load and stress testing of Web servers; deliver browser-based test management; and
overall Web access.
Competition
SQA differentiates itself from the competition by its strategically deﬁned focus on cross-Windows client-server application testing and by its suite of tools that are integrated via an
underlying test repository. Other testing tools competitors, such as Mercury Interactive and Segue, include Windows testing capabilities in their core products, but not the extent of support
that SQA Suite currently supplies. This concentration has allowed SQA to deliver solutions to Windows-speciﬁc problems not addressed as rapidly by the other vendors.
As mentioned above, SQA Suite also provides unique capabilities to Windows application testers through its ability to leverage testing scripts across multiple versions of Windows,
(including 16- and 32-bit versions), and to test application components such as VBX, OCX, and other objects.
What SQA Suite Users Have To Say
Users interviewed by Aberdeen lauded SQA Suite’s underlying SQL-based repository which allowed them to automatically generate in-depth reports of the test results, were greatly beneﬁted
by the defect tracking mechanism supplied by SQA Manager, and found SQA’s tools the most ﬂexible for importing existing manual test data from other sources. Users were unanimous in
their support of automated testing and overall received a fourfold improvement in test coverage, length of test cycle, and types of tests executed. In addition, all users contacted agreed that
SQA’s test planning capabilities and Windows GUI interface made QA testers more productive, both by better test design up front and by allowing even non-programmers to be effective
testers through the SQA point-and-click interface.
This section will discuss a several speciﬁc customer experiences and opinions of SQA Suite and SQA’s testing tools in general.
One brokerage ﬁrm is using SQA for regression testing on a hybrid environment that runs in an X Windows session and on Windows. While no product is speciﬁcally geared for this market,
SQA Suite was chosen for its defect tracking capability and tight integration which allowed testers to move from script writing right into defect tracking .
Having used a manual testing process previously, automated testing has resulted in a signiﬁcant time savings that’s been enhanced by SQA’s structured methodology. Testers working on the
ﬁrst pilot plan walk through 70 procedures with 400 test cases that would take them two weeks to do manually. Automated by SQA, the testing process now takes one day—providing there is
no signiﬁcant maintenance required on the scripts. Even in the case of script maintenance, testing on this pilot application would take a maximum of three days, claims the SQA user, and
that’s a savings of seven days magniﬁed by the 40 or so more iterations that will have to be tested before the application is ﬁnished.
Finally the user is enamored of SQA Manager which allows him to maintain an organized and modular approach to testing that also enables him to quantify and keep track of what testers are
doing as a group. With other automated testing tools, defect tracking and management tools had to be purchased separately. This user was sincerely pleased with the integrated SQA approach
to automated testing.
Another SQA user at a telecommunications company found that SQA Suite’s low overhead on the network was a plus and was impressed that SQA could interpret a object wrappers placed
around COBOL code to create a VBX and give the tester the correct object states and information. This tester also found the product more comprehensive that other tools out on the market.
Automated testing was the only way that this tester could support a demanding product delivery schedule.
Technical support is an important issue for this SQA customer who received good response from SQA at all times.
One large oil and gas exploration ﬁrm is currently using SQA to test a series of legacy and 32-bit software applications that will be eventually deployed on both Windows 95, Windows NT
workstation 3.1, and Windows NT. SQA Suite’s ability to test multiple versions of Windows, both 16- and 32-bit was certainly a factor in this user’s decision to use the test tool. However, the
single biggest attraction was SQA Suite’s ease of use together with the packaged suite of tools. SQA Robot’s scripting language, so close to BASIC, was also a bonus, since several
developers were already familiar with Visual Basic and could leverage this knowledge to create test scripts.
SQA’s database repository, especially the addition of a full-ﬂedged SQL database, is reliable, robust, and scalable to a large number of developers, said this user. Previous testing tools used
by this SQA customer produced database capabilities that were limited in scope. In one instance performance degraded signiﬁcantly and data was corrupted with only six developers hitting
the database at one time.

Conclusions
The increasing emphasis on delivering quality software in a timely manner has also increased the importance and demand for quality testing tools. SQA has carved out a signiﬁcant niche for
itself by meeting this demand in the Windows market, providing broad functionality, and delivering the testing tools wrapped in a methodology to help developers plan, design, and manage
the testing process. A bonus of this process is the ability for developers to make fact-based decisions regarding the release and quality of software, thereby eliminating the guesswork and
resulting blunders that follow the deployment of buggy and unscalable systems.
As one of the leaders of the Windows testing tools market, SQA will face challenges over the next two years to keep ahead of the existing competition and those that will enter the crowded
market. Aberdeen believes that SQA will need to integrate with application design tools in order to extend the test cycle to the design phase; complete a suite of Internet testing tools; and
beef up load testing support with the ability to emulate multiple users from one PC or SMP machine. Aberdeen expect s that SQA will do this and maintain a ﬁrm standing as an innovator
and leader in cross-platform Windows testing tools.
Testing is now a vital piece of the Windows client-server application development cycle and SQA an excellent means to implement it. Successful testers will note that SQA Suite delivers
support, via tools and methodology, for all the stages of successful testing. Aberdeen suggests that IS managers and software developers, who have not already done so, should start looking
for comprehensive and integrated testing tools immediately, giving SQA Suite 5.0 a high priority.
