Preface
The hot 1996 issue of which platform, IBM’s AS/400 or Microsoft’s NT Server
with BackOffice, is best for running production, bet-your-business applications
has been settled for most AS/400 users — AS/400 won hands down in terms of
reli-ability, scalability, supportability, total cost of acquisition and maintenance,
MIS staff costs, return on investment, and probable business disruption. (See
Aberdeen’s Viewpoint, IBM’s AS/400 Versus Microsoft’s NT Server: The
Challenge That Is Not, January 1996).
Now, in February 1997, IBM and Microsoft are quietly telling the world how
they have come to make peace together in the AS/400-NT Server product
space. The messages are low key. Both suppliers’ senior management want
zealot-advocates for each approach of computing to "cool it" and build upon
the other’s strengths. Real MIS executives — those individuals charged with
guiding their enterprises’ technology choices — can make the most rational
decisions when they have the facts, not the hype. This Profile describes the very
positive status of the IBM AS/400-Microsoft relationship today.
Executive Summary
IBM announced early in 1996 that it had begun talking with Microsoft to
resolve how it could deliver to its customer base the benefits of NT Server in
the same integrated way that it was delivering LAN Server/400. Despite initial
mis-starts, IBM and Microsoft reached an agreement in late 1996 on how NT
Server, Microsoft’s network operating system (NOS), will be integrated into
IBM’s AS/400 platform.
The bottom line is that NT Server 4.0 is scheduled to be deliverable on IBM’s
AS/400 Integrated PC Server and generally available for AS/400 users during
1Q98.
IBM believes that the greatest applicability for NT Server on the Integrated PC
Server will be in satellite offices with a small number of users where AS/400 is
the chosen platform for production applications, NT Server the selected NOS,
and users want to move data from their production AS/400 applications to their
desktop Microsoft Office for NT or Lotus SmartSuite applications. In this
environment, users will still be using their AS/400 production applications that
integrate with the enterprise’s central system while using NT Server for file and
print functions that support desktop PCs. This eliminates the need for a separate
PC server on a LAN with all the costs and management issues associated with
two operating environments — issues that are intensified in a smaller site.
AS/400 users will now have their choice of Novell NetWare, IBM LAN
Server/400, and Microsoft NT Server for a network operating system running
on the Integrated PC Server in their AS/400. With these investments, IBM
continues to prove its ability to open the AS/400 to meet its customer base’s
requirements.
In addition to being able to run integrated desktop office application suites on
NT, users will have the ability to run NT Server’s BackOffice software. This
suite of tightly integrated packages — such as SQL Server, Exchange, and
Internet Exchange — is necessary to upgrade NT Server from a NOS to a
production application platform.
However, NT Server is not as tightly integrated with OS/400 as LAN
Server/400 and NetWare are. Even running on the Integrated PC Server it will

require its own operator console and will not be able to use the AS/400’s. And
even with this limitation, the benefits for AS/400 users are significant.
If requested, IBM will support NT Server on the Integrated PC Server hardware
everywhere and to the same high quality standards with which it services the
AS/400. This will aid MIS managers charged with keeping far-flung distributed
systems operational at all time. When problems arise, IBM — or a local IBM
business partner — will be ready to handle them.
IS managers will be able to use the AS/400s common, shared disk environment
for NT and to initiate back-ups. This will decrease the biggest cost of all for
operating a NOS — internal operations support costs.
End-users will have a common profile for both OS/400 and NT. This will
simplify their administration requirements — change a password on the profile
and it will apply to both NT and the AS/400. In addition, users can transfer data
from DB/2400 to their NT applications through the use of the ODBC (open
database connectivity) standard. This will prove a real benefit for such user
requirements as generating mass mailings from a customer database or
developing customized presentations for prospective insurance customers.
However and most importantly, in satellite locations where employees should
be doing the jobs that their enterprise needs them to do — whether it be to sell
products or process claim forms or make widgets — this product will lessen the
burdens on those non-MIS office staff members responsible for maintaining the
local systems on a daily basis. This is a real benefit for real users.
The Power Of The Integrated PC Server
The AS/400 has been transformed over its 20+ year history — starting out as
the System/3 — to meet its customers’ changing computing requirement. Major
architectural changes have resulted in product name changes — functionality
upgrades in progressively newer models.
A major challenge to advancing the functionality of the AS/400 has been in the
area of providing NOS services for attached PCs. The AS/400s design center is
for running production online transaction processing, batch and decision
support applications while PC LAN capabilities focus on file, print, and
communications services. Two very different functionality specifications yet
both are required by today’s PC-empowered knowledge enterprise worker. In
the past, such a requirement might have led to a whole new AS/400-System/3X
architecture. Today, it challenged IBM’s engineers to develop an enhancement
for the AS/400.
IBM’s innovative solution to this user need to access production AS/400
applications from a PC LAN environment while minimizing MIS support costs
was to add a NOS into the AS/400. This was a two-part engineering task. First,
IBM needed to design a hardware add-on that could most effectively run a
NOS. Second, IBM needed to modify any targeted NOS and make extensions
to OS/400 in order that the NOS could run seamlessly within the OS/400
operating environment.
To run NT, IBM is providing an Intel Pentium Pro 200 MHz processor-based
PC server board with 512 Mb of memory that can fit into an AS/400 slot. Disk,
tape, and I/O services are provided by the AS/400 itself — the board primarily
provides the compute and dedicated memory capabilities. This PC-card was
originally named the FSIOP (file server input/output processor), pronounced a
tongue-twisting "fizzy-op." But has now been renamed with the more
descriptive moniker, Integrated PC Server.
Adding software to run on the processor that can be effectively managed by
OS/400 is a more complex undertaking. IBM originally designed the Integrated
PC Server to run its OS/2 operating system. And the AS/400 division was able
to quickly port OS/2 LAN Server to the Integrated PC Server and integrate it
with OS/400. Next, working in conjunction with Novell, NetWare for OS/2 was
added as an option. And the AS/400 division has announced that Domino 4.5
— the new web-enabled version of Lotus Notes — will be available in May,
1997. In the future, the AS/400 division will run Domino in native mode on the
AS/400 — and this function is expected to be available in native form at year
end 1997.
All things considered, the Integrated PC Server has more than met the
objectives IBM and the user base originally had for it. And users report that its
use creates insignificant degradation in AS/400 performance for running
production applications — thus surpassing the anticipated negative result of
porting a NOS directly to the AS/400’s CPU.
Addressing the Demand For NT Server As A NOS
Due to the dominance of Microsoft’s family of Windows operating systems on
the desktop and popularity of Microsoft Office for desktop applications,
AS/400 users requested IBM to provide them with NT Server on the Integrated
PC Server. This was an especially important requirement for enterprises that
had selected NT as a corporate-wide NOS standard.

The key question IBM needed answered was how much support Microsoft
would provide to meet the user requirements for ease of management on the
AS/400.
Microsoft has a corporate policy not to modify NT Server for any hardware
manufacturer. The philosophy is that NT should be NT — no matter what brand
of server it is residing on. While the objective is to maintain consistency and
avoid the numerous permutations and variations that are the norm in the Unix
world, it also limited IBM’s options. The result is that NT Server on the
Integrated PC Ser-ver will require a separate management console and will not
be able to be managed directly through AS/400 as comprehensively as other
NOSs now available.
The points of integration between OS/400 and NT Server are ODBC (the
industry’s Open Data Base Connectivity standard), common user profiles, and
integration of the OS/400 and NT file systems. Print integration services will be
available with the next release of OS/400.
With these capabilities, users can easily run Office for NT on the Integrated PC
Server and transfer data from DB2/400 to Office applications. And use the disk
storage and back-up facilities of the AS/400.
For those MIS managers who want to incorporate IBM’s new Network Station
— the network computer — with their AS/400 systems, the use of Office for
NT on the Integrated PC Server will allow users to access their familiar desktop
applications with as much speed — if not more — than on their older Pentium
desktops.
While MIS managers could also put Microsoft’s BackOffice Suite of
transaction-oriented applications, such as SQL Server, Internet Server, and
Transaction Server, on the Integrated PC Server, this will probably not be the
chosen direction. The AS/400 has the more robust production qualities — such
as reliability, serviceability, availability, security — required for transaction
processing. And MIS managers who are determined to run transaction-based
applications on NT Server and BackOffice will probably decide that two 4-
processor PC servers with availability clustering capabilities are a better
platform choice.
IBM — Delivering What Its Customers Want
IBM has had to overcome numerous technology and marketing obstacles to get
NT Server on the Integrated PC Server for its AS/400 customers. It has done so
and should be congratulated for the determination it has demonstrated.
According to the current schedule, NT Server will be available as an option
from IBM’s Business Partners and IBM on the Integrated PC Server sometime
during the first quarter of 1998. This will allow MIS executives the time
necessary to plan for budgets and roll-out schedules while IBM and Microsoft
technical staff ensure the product’s integrity.
IBM believes that its first customers for NT Server in the AS/400 will be the
distributed, satellite offices of larger organizations. Such enterprises have two
issues that need addressing from an MIS perspective. One, the satellite office
should be running the same operating environment — applications and other
technology components — as are used at the central site and are corporate
standards. Two, without access to the same professional MIS resources
available at the central location, both tools and procedures need to be employed
that can maintain the same service levels and at the same low cost as are
achieved at the main location. This dual requirement for standardization and
simplicity with the objective of lowering total MIS costs — including the costs
involved in training users on how to effectively use the organization’s MIS
capabilities — will pull NT on the Integrated PC Server into the majority of
installations where it is anticipated to be used.
As it has in the past, IBM will continue to increase the performance of the
Integrated PC Server. And multiple Integrated PC Servers can be added into an
AS/400 to increase the number of users supported. The result will be IBM’s
ability to provide a smooth growth path for organizations as they add NOS
users to their AS/400.
But most importantly, IBM and its Business Partners will support the physical
in-stallation of NT on the Integrated PC Server anywhere in the world — just
like they do the AS/400. However, note that if an NT-related problem arises,
customers will be encouraged to seek assistance from Microsoft.
Finally, Aberdeen notes that the Integrated PC Server is unique in the server
mid-range — no other major suppliers have designed a system in this integrated
man-ner with a bank of main processors dedicated to running enterprise-core
business applications while a different processor set in the same cabinet
processes file and print services. IBM has shown a great deal of creativity and
ingenuity in bringing this design forward to solve its customer’s systems
integration problems.
Conclusions

IBM, Microsoft, and MIS executives that want both the AS/400 to run their
production applications and NT for their NOS are delighted about the current
plans for placing NT on the Integrated PC Server. While greater integration
with OS/400 would have been desirable, this capability is far superior to
requiring users of both technologies to maintain two separate servers on
individual LANs.
A serious concern is that this planning announcement may cause rumblings
from industry kibitzers that AS/400 and NT should be more tightly integrated
and provide more application interoperability services or that this is just a first
step for NT to replace AS/400 as the production platform-of-choice for
businesses. But the reality is that it helps MIS executives manage a combined
production application and NOS environment — especially in satellite
locations where MIS skills are scarce and relatively expensive — in a
pragmatic and elegant fashion. And this is what real users want and need.
One of the excitements at IBM Corporation is that the combination of the
AS/400, NT, Office for NT or Lotus SmartSuite, and Network Stations will
provide users with the real benefits of Network Computing that the senior IBM
executives have been describing throughout 1996. From a fast, low-cost
desktop device users will be able to run AS/400 production applications in one
window, desktop applications in a second and move data from the production
application to the desktop one — all the while being online with the
enterprise’s intranet. And MIS will be able to manage this relatively complex
user environment at a significantly lower cost than today’s separate PC LAN
and AS/400 operations.
Enterprises, MIS executives, end-users, IBM, and Microsoft are all winners
from the porting of NT Server to the Integrated PC Server.

### Abstract

Aberdeen Group documents the IBM-Microsoft agreement of late 1996 to integrate NT Server 4.0 into the IBM AS/400 Integrated PC Server, scheduled for general availability in Q1 1998. The study explains the technical architecture (Intel Pentium Pro 200MHz board in AS/400 slot, sharing disk/tape with OS/400), identifies the integration points (ODBC, common user profiles, file system integration), and concludes that both IBM and its AS/400 customers benefit substantially from the arrangement, enabling enterprise users to run AS/400 production applications alongside NT-based desktop applications in a unified, cost-efficient infrastructure.
